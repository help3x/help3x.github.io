  • PostCSS を始める

    • インストール

      npm install postcss-cli --save-dev

    • PostCSS Preset Env

      • 最新の CSS をブラウザが理解できる形式に変換する。

        npm install postcss-preset-env --save-dev

    • 使い方

      バージョンを表示する。

      .\node_modules\.bin\postcss --version

      input.css を変換し output.css として出力する。
      プラグインとして「postcss-preset-env」を使う。

      .\node_modules\.bin\postcss input.css --use postcss-preset-env --output output.css

      複数のプラグインを使う。

      .\node_modules\.bin\postcss input.css --use postcss-preset-env --use autoprefixer --output output.css

      cssnano を使って圧縮する。ソースマップも出力しない。

      .\node_modules\.bin\postcss input.css --no-map -u postcss-preset-env -u autoprefixer -u cssnano -o output.min.css

      parser に SugarSS を使う。

      .\node_modules\.bin\postcss input.css --no-map -u postcss-preset-env -u autoprefixer --parser sugarss -o output.css
      .\node_modules\.bin\postcss input.css --no-map -u postcss-discard-comments -u postcss-preset-env -u autoprefixer --parser sugarss -o output.css
      .\node_modules\.bin\postcss input.sss --no-map -u postcss-discard-comments -u postcss-preset-env -u autoprefixer --parser sugarss -o output.css

      Minify

      .\node_modules\.bin\postcss input.sss --no-map -u postcss-discard-comments -u postcss-preset-env -u autoprefixer -u cssnano --parser sugarss -o output.min.css

要約

要約練習におすすめの字数が 100 文字。 口頭で読むと 15～20 秒程度。

1000文字程度の新聞の社説を使って練習する。 次の3点を押さえる。

  • 何について語っているか（話題）
  • 書き手はどう考えているか（結論）
  • どうしてそう考えるのか（根拠・理由）

読み終えた本の感想を書く。 次の3点にしぼって100字にまとめる。

  • 何についての本なのか（話題）
  • この本の面白い点はどこか（見どころ）
  • 自分はどう感じたか（感想）

100字要約に慣れたら、200字要約も練習してみる。 こちらは文章を構成する力が付く。

字数を増やすときに意識する 3 要素

  • 対比（「Ａは～だが」「これまで～だったのに対し」など。譲歩（「仮に～しても」も））
  • 理由（「～ので」「～から」。「～するため」などの目的も）
  • 例（「たとえば～など」「～のように」）

字数を減らすときには、逆にこの 3 要素を削ることを意識する。

40字程度の要約。

まとめ見出し作り。13字を目安に。

見出しの目的によって、次の点を使い分ける。

  • 結論を隠す／隠さない
  • 注目するよう煽る／淡々と伝える
  • 専門用語が通じる前提／一般向け
ログライン

映画用語で、映画のあらましを簡潔に表現した一文のことを言う。

細部や脇のエピソードを省き、本筋だけを示す。

定番の型 「Ａな主人公が、Ｂに出会い、Ｃをする」

（Ａ）「正義の味方のアンパンマン」が、（Ｂ）「世界征服をもくろむばいきんまん」を（Ｃ）「やっつける」。

（Ａ）「家族愛を知らない女子高生」が、（Ｂ）「大家族」に居候し、（Ｃ）「自信と夢を見出す」。

エッセイやビジネス書、実用書の感想を書く場合、それらの本をまとめる場合。 これらの本の説得力や魅力は、次の 3 つの要素から成り立っている。

  1. 誰が語っているか（WHO）

    たとえば、羽生結弦が書いた本。

  2. 何を語っているか（WHAT）

    著者は無名であっても、その本でしか読めない貴重な内容が書いてあれば読むはず。

  3. どんな観点から語っているか（HOW）

    昔から論じられているテーマであっても、新しい切り口で語られれば、新鮮な気付きが得られそう。

（例）

【誰が】予備校の国語講師が 【何を】ビジネスパーソンの文章術を 【どんな観点から】身近で親しみやすい例文を用いて解説する

作品概要をこれら3点でまとめたあとに、次の2点を書けば感想文ができあがる。

  • その中での見どころ
  • 見どころに対する自分の感想

追加情報として次の内容を書けばさらに親切。

  • どういう人におすすめか
  • 注意事項は何か（上級者には物足りないかもしれない、など）
トレーニング
「○○どうだった？」と聞かれたら、「ＡがＢに出会い、Ｃをする」の型、もしくは「誰が、何を、どんな観点から」の型で説明してみよう。
