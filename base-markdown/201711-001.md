## セレクタの書き方

[属性名]
:   属性名を含むものが対象。  
    `a[target]`と書けば a要素でtarget属性を持つものが対象になる。

<!-- -->

[属性名="値"]
:   属性名に値が指定されたものが対象。  
    `a[target="_blank"]`と書けば、a要素でtarget属性に「_blank」が指定されたものが対象になる。

<!-- -->

[属性名*="値"]
:   属性名に値を含むものが対象。  
    含む＝部分一致で探す。
    `a[target*="blank"]`と書けば、a要素でtarget属性に「blank」を含むものが対象になる。  

<!-- -->

[属性名~="値"]
:   属性名に値が複数指定されているとき、その中に値で指定されたものがあるものが対象になる。
    含むものではなく完全一致するもの。

    ```html
    <div class="abc def">×</div>
    <div class="def ghi">○</div>
    <div class="def ghijk">×</div>
    ```

    ```css
    [class~="ghi"]
    ```

    この場合は二番目のdivのみ、スタイルが適用される。
    三番目のdivは「ghi」を含むが完全一致ではないので適用されない。

<!-- -->

ほか。  
[属性名^="値"] は前方一致。  
[属性名$="値"] は後方一致。
[属性名|="値"] はハイフン区切りの値をとり、その値が値で指定されたもので始まるもの。

[lang|="ja"]
だと
`<html lang="ja">` は適用される。  
`<html lang="ja-jp">` は適用される。  
`<html lang="japan">` は適用されない。

### 命名規則

BEM
:   要素をBlock、Element、Modifierのどれかに分類して命名する。 \
    BEMの名前は、それぞれの頭文字を取ったもの。

Block
:   構成のルートとなる要素。 \
    たとえばボタン、グリッド、タブ、見出し、画像など。 \
    この要素を基点として考え、サイトはBlockを組み合わせることで構成する。

Element
:   Blockを構成する一つ一つのパーツ。Blockでしか生きられない。 \
    たとえば、グリッド内のセル、タブのボタンやコンテンツ、見出しや画像など。

    BEMではBlockから見て、下の階層にいる要素はすべてElementとして扱う。

Modifier
:   BlockとElementのバリエーションや状態の変化をつくるときに指定する。 \
    たとえば、Block（.box）に背景色をつけたパターンを作る場合、Block自体ではなく、背景色を持たせるModifier（.box--hilight）を作る。

    ```css
    .box {
      display: block;
    }
    .box--hilight {
      background-color: gray;
    }
    ```

    スタイルをリセットしていくより、足していくようにすると管理しやすくなる。


MindBEMding
:   BEMをCSSのクラス名に適用するために作られた規則。 \
    Elementはアンダースコアを2つ、Modifierはハイフンを2つで連結し、各要素を表現する。

    ```css
    block
    block__element
    block__element--modifier
    block--modifier
    block--modifier__element
    ```

    ```html
    <div class="block">
      <div class="block__element"></div>
      <div class="block__element block__element--modifier"></div>
    </div>
    <div class="block block--modifier">
      <div class="block__element block--modifier__element"></div>
    </div>
    ```

#### BEMの基本ルール

##### クラスセレクタ単体に指定する。

IDセレクタや要素セレクタは使わない。

##### ElementとModifierを単体で使わない。

ElementとModifierは親となるBlockの中だけで指定できる。  
たとえば、ある箇所のデザインが`.box__element`と同じスタイルだから`.box__element`だけ使いまわす。というのはしない。

```html
<div class="foo">
  <!-- NG: 親要素がfooなのに、boxのスタイルを適用している -->
  <div class="box__element"></div>　
</div>
```

#### よくある問題

##### Element__Element

HTMLの構造をElementによって表す必要はない。子孫要素が多くなったとしても、`.block__element__element`のようなクラス名にはしなくてよい。  
でないと、階層が深くなるにつれてクラス名が長くなってしまう。

```html
<!-- NG -->
<div class="box">
  <div class="box__element">
    <div class="box__element__element">
    </div>
  </div>
</div>
```

```html
<!-- たとえば、こんなのでもOK -->
<div class="box">
  <div class="box__element">
    <div class="box__child-element">
    </div>
  </div>
</div>
```


##### Block__Element--Modifier

すべてのElementにModifierを指定するのは冗長で、指定忘れが出る可能性もある。

```html
<!-- NG -->
<div class="box">
  <div class="box__element box__element--pad-large"></div>
  <div class="box__element box__element--pad-large"></div>
</div>
```

これを改善するには、Block--Modifierを起点にスタイルを指定する方法がある。

```html
<div class="box box--pad-large">
  <div class="box__element"></div>
  <div class="box__element"></div>
</div>
```


#### Blockの粒度

##### コンテナとコンテンツの分離（OOCSS）

コンテナとは`.container`や`.wrapper`のような横幅を制限したり、グリッドレイアウトのような大枠のレイアウトを指定するもの。  
コンテンツとは本文やリスト、画像やテーブルなどのパーツのこと。

このデザインにはどんな構造（レイアウト）が必要なのか。その中にはどんなパーツがあるのかという順番で考えていくと作りやすい。


##### どこまで汎用性を持たせるか

Modifierの組み合わせが複雑になってきたら、別のBlockとして定義できないか考える。  
「このパターンではこのBlockに2つのModifierを指定してください」よりも「このパターンではこのBlockを指定してください」の方がシンプル。


#### 設計手法の組み合わせ

BEMとそれ以外の設計手法を組み合わせることもできる。  
たとえば、

* OOCSS
* SMACSS
* 汎用クラス（ユーティリティクラス）

などの設計手法がある。

BEMだけですべてを解決することは難しいので、ほかの設計手法を柔軟に取り入れるのがベターなやり方。





## FLOCSS

### 基本原則

Foundation、Layout、Objectの3つのレイヤーと、Objectの子レイヤーで構成される。

1. Foundation
2. Layout
3. Object
    #. Component
    #. Project
    #. Utility


#### Foundation

ブラウザのデフォルトスタイルの初期化や、プロジェクトの基本的なスタイルを定義する。
初期化にはReset.cssやNormalize.cssなどを用いる。


#### Layout

ページを構成するヘッダーやメインのコンテンツ、サイドバーやフッターといったプロジェクト共通のコンテナーブロックのスタイルを定義する。


#### Object

OOCSSのコンセプトを元に、プロジェクトにおける繰り返されるビジュアルパターンをすべてこのレイヤーに定義する。

Objectレイヤーは、さらに次の3つのレイヤーに分けられる。

クラス名にはプレフィックスを付ける。

| レイヤー  | プレフィックス |
|-----------|----------------|
| Comopnent | .c-** |
| Project | .p-** |
| Utility | .u-** |

##### Component

再利用できるパターンとして、小さな単位のモジュールを定義する。  
できるかぎり、最低限の機能を持ったものとして定義されるべきであり、それ自体が固有の幅や色などの特色を持つことは避けるのが望ましい。

たとえば、BootstrapのComponentカテゴリに見られる button などが該当する。


##### Project

プロジェクト固有のパターン。  
いくつかのComponentと、それに該当しない要素によって構成されるものを定義する。

たとえば、記事一覧、ユーザープロフィール、画像ギャラリーなどのコンテンツを構成する要素が該当する。


##### Utility

ComponentとProjectのレイヤーで解決することが難しい、または適切ではない、わずかなデザイン調整のためのスタイルを定義する。  
clearfixのためのルールセットが定義されているヘルパーも、このレイヤーに含める。

Object自体が持つべきでない margin の代わりに `.mbs { margin-bottom: 10px; }` のようなスタイルを用いて、隣接するモジュールとの間隔をつくるといった役割を担う。





## Vertical Rhythm（縦のリズム）

縦の空間を一定の間隔で分割し、それの倍数になるようにデザインする方法。


### 一定の間隔（ベースライン）の決め方

ベースラインは`line-height`によって決定される。  
たとえば本文の`line-height`が24pxだとすると、ベースラインは24pxになる。

ということで、次の2つのルールを使ってVertical Rhythmを適用する。

#. 要素間の縦の空白を24pxの倍数（または2で割ったもの）に設定する。
#. すべてのテキスト要素のline-heightを24pxの倍数（または2で割ったもの）に設定する。


Note:  
一例として24pxを使っただけで、必ずしも24pxしなければならないということではない。  
18pxでも何でもかまわない。
が、必ずその倍数を指定すること。

## リセットCSSとノーマライズCSS

リセットCSS
:   デフォルトスタイルシートで定義されているスタイルをリセットし、ブラウザ間の差をなくすCSS。 \
    フォントの規定値や余白の設定など、すべてのデザインをゼロベースでやるときに使う。

ノーマライズCSS
:   ブラウザ間の差をなくすCSSだが、リセットCSSとは異なり、デフォルトスタイルシートで有用な部分はそのまま使われる。 \
    差がある部分のみ調整するCSS。





LV（ライトバリュー）
:   実際の光の量（明るさ）を表す。

LV15の明るさが適正だとしたとき、EVも同じEV15にすれば適正露出で撮影できると考えてよい。
