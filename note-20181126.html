BULK COLLECTとFORALLによるバルク処理

BULK COLLECTについて

バルク処理を利用して問合せを行うには、BULK COLLECTをINTOキーワードの前に置いて、1つ以上のコレクションをINTOキーワードの後に指定します。BULK COLLECTの動作方法について知っておくべきことは次のとおりです。

  • 3つのコレクション型（連想配列、ネストした表、VARRAY）のすべてで使用できる。
  • フェッチした情報を個々のコレクション（SELECTリスト内の式ごとに1つ）に格納することも、レコードの1つのコレクションに格納することもできる。
  • コレクションは常に、インデックス値1から開始し、密に移入される。
  • 1行もフェッチされなかった場合、コレクションからすべての要素が削除される。

FORALLについて

ループ内でDML文を実行するときは常に、FORALLを使用するようにそのコードを変換してください。その変換によるパフォーマンス向上に開発者は驚き、ユーザーも満足するでしょう。

FORALL文はループではなく、PL/SQLエンジンに対して次のことを伝える宣言文です。

“通常なら1行ずつ実行されるDML文をすべて生成し、それらのDML文を1回のコンテキスト・スイッチでSQLエンジンに送信する”

FORALL文の“ヘッダー”の見た目は数値によるFORループと同じですが、LOOPキーワードやEND LOOPキーワードはありません。

FORALLについて知っておくべきことは次のとおりです。

  • 各FORALL文には1つのDML文のみを記述できる。ループに2回の更新と1回の削除が含まれる場合は、3つのFORALL文を記述する必要がある。
  • PL/SQLでは、FORループの場合と同様に、FORALLイテレータ（リスト4の30行目のindx）を整数として宣言する。このイテレータと同じ名前の変数を宣言する必要はなく、また宣言すべきではない。
  • DML文の少なくとも1か所で、コレクションを参照して、そのコレクションのインデックス値としてFORALLイテレータを使用する必要がある（リスト4の35行目を参照）。
  • FORALLヘッダーでIN low_value . . . high_value構文を使用する場合、FORALL文の内部で参照するコレクションは密に入力されている必要がある。つまり、low_valueからhigh_valueまでのすべてのインデックス値が定義済みである必要がある。
  • コレクションが密に入力されていない場合は、FORALLヘッダーでINDICES OF構文またはVALUES OF構文を使用する必要がある。

登録更新削除の判定

削除

次のいずれかを満たすものを削除と判定する。

  • 編集状態が「NDE」または「UDE」である。
  • 既存レコードで、主キーの値が変更されている。

登録

次のいずれかを満たすものを登録と判定する。

  • 編集状態が「ADD」である。

  • 主キーの値が変更されている。

    ただし、変更後のキーと同じレコードが存在したらキー重複エラーとする。

更新

次のいずれかを満たすものを更新と判定する。

  • 編集状態が「UPD」で、主キーの値が変更されていない。

  • 主キーの値が変更されていて、変更後の主キーの値に該当するデータが存在する。

    というわけでこのパターンはキー重複エラーにします。

課題 これだと複数人が同じタイミングで保存したとき、後勝ちになってしまう。 ほかのマスタメンテは、行追加したデータに該当する内容がDBに存在したときはエラーにしている。

一人で使ったとき ブラウザ側は、主キーが重複していればエラーとし要求を送信しない。 サーバー側は、主キーが重複していたら後勝ち。それともコレクションの中に同じ主キーがいたらエラー？　最初のやつが優先？

v8での破壊的変更（GitHubのリリースノートより）

  • 単純なリセットやユーザー定義のスタイルを使用するために、すべての基本要素スタイル（本文、タイポグラフィ、フォーム、および表スタイルを含む）が削除されました。移行を助けるために、新しいbasscss-basicパッケージをそのまま使用するために、またはカスタム基本スタイルの出発点として使用するためにまとめました。
  • btn、btn-primary、およびbtn-outlineモジュールは、コアパッケージから削除されました。オプションでbasscss-addonsで利用可能です。
  • カラーフォームモジュールがコアパッケージから削除されました。新しいオプションのbasscss-formsモジュールを置き換えることができます。
  • すべてのカラースタイルがコアから削除され、ユーザー定義スタイルとより個別のカラースタイルモジュールが採用されました。インスピレーションのためにいくつかのカラーモジュールがbasscss-addonsまたはcheckout basscss/themesで利用可能です。
  • フレックスオブジェクトモジュールは、より一般的で強力なユーティリティスイートであるbasscss-flexboxに置き換えられました。
  • .containerスタイルはbasscss-gridから削除され、basscss-layoutのmax-widthユーティリティーに置き換えられました。
  • basscss-borderモジュールには、境界色のスタイルは含まれません。デフォルトでは、罫線は前景色を継承します。色を調整するには、境界色のユーティリティを使用します。
  • basscss-responsive-statesモジュールは、basscss-hideモジュールに置き換えられました。

コアから削除されたモジュールはbasscss/addonsから入手できる。これらのモジュールを使うには、コアモジュールに加えてアドオンのモジュールをインポートします。

説明

max-width
表示領域の最大幅。ここに指定した横幅以下のときに適用される。
min-width
表示領域の最小幅。ここに指定した横幅以上のときに適用される。

例 

@media (max-width: 480px) {
  /* 480px 以下の場合 */
}

@media (min-width: 770px) {
  /* 770px 以上の場合 */
}

@media (min-width: 640px) and (max-width: 800px) {
  /* 640px ～ 800px の場合 */
}
